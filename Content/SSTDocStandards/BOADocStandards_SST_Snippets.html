ActionableReports

NOTE: Actionable reports are error reports with a non-zero record count. Report types for error reports are:

Error—Identifies reports that display data that must be fixed before it can be loaded into the target system.Target Readiness—Identifies reports with data that cannot be loaded into the target system, such as a record missing required fields, invalid check table values, or a field with an incorrect number of decimal places. These records will either produce an error on load or load incorrectly. Records on these reports are probably excluded from the target export.Business Readiness—Identifies reports with data that if loaded into the target system would decrease data quality or cause business errors in the data. This data could load into the target system (as in, loading this data would not cause errors). However, the data should not be loaded into the target system because it violates business rules.

In Migration Reports and Transform mention the Actionable report type. 

Add Mode

If no records exist, the page displays in add mode. Otherwise, click Add.

Use in steps when a page opens with a blank row ready to add data.

To add a scenario to a business process, you must configure security to include the position for the business process. Refer to Configure Business Processes in a Position for more information.

Quality Dimension

Displays a recognized term used by data management professionals to describe a feature of data that can be measured or assessed against defined standards to determine data quality. Refer to Register Quality Dimensions for more information.

NOTE: All Error reports require a quality dimension.
