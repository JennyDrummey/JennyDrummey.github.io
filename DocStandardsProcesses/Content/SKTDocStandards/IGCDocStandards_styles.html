SKT Styles

Use the default style sheet in Flare. Generally accepted technical writing style standards for writing use cases apply.

  • Use bold text (only) for options (links, icons, etc) that are a recipient of a user action in a use case

  • Italics used for page names, as in, asset details page.

  • There are no Page Descriptions

  • For H1s, use capitalization case. For all headings after H1, capitalize first word and special terms only.

For example:

This is the First Heading

This is the second heading

This is the third heading

Click Quick Add.

NOTE: To access the term after you click Quick Add, search for the term.

You can refer to the Sponsors topic in the help as a good use case example.

In-line Formatting

In-line formatting should be avoided when it can be in favor of applying styles defined in the css file. However, sometimes you must use <b></b> and <i></i> span tags. To ensure that these spanned styles display correctly in browser outputs, remove spaces inside the spans. For example: spaces inside spans like <b>NOTE: </b> should be updated to <b>NOTE:</b>

What's New Table

  • Use the td.date style for the text in the column on the left.

  • Use TableText for all text in the column on the right.

  • Bold the Enhancement or Bug Fix heading in that column
