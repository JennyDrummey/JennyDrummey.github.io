UI Terms for the SKP

Use the following guidelines when referencing items on the User Interface of Syniti.

Term

 Location/Description Comment

Profile menu

Upper right menu on Site bar.

Displays user name, tenant name, What’s New, My Account, Admin

Always italicize menu names, and lowercase the word menu, whether in steps or in use cases.

Example:

Click My Account on the Profile menu.

Home menu

Displays module names to the right of the Syniti logo

Always italicize menu names, and lowercase the word menu, whether in steps or in use cases..

Example:

Click Migrate on the Home menu.

Site menu

In the upper right, with the Notifications bell, page help etc.

Always italicize menu names, and lowercase the word menu, whether in steps or in use cases..

Example:

Click the Notifications icon on the Home menu.

check box

Clickable element that selects the option description for use.

  • Bold the name of the check box in steps

  • Do not bold the name in use cases, but capitalize the name as it appears in the UI

  • Do not capitalize the words check box

list box

Clickable element that expands and displays a list that users can make selection(s) from.

  • Bold the name of the list box in steps

  • Do not bold the name in use cases, but capitalize the name as it appears in the UI

  • Do not capitalize the words list box

Incorrect nomenclature for this element includes:

  • drop-down
  • drop-down menu
  • drop-down list
  • drop down

modal

 

Modal window that opens in Orchestrate when adding a task, for implementations when adding a query dataset.

  • Bold when in steps, e.g., Click the Create Query Dataset modal.

  • Italicized when referencing the page name in non-steps, e.g., You can view details on the Add a Recurring Schedule modal.

  • Do not add the word "window"

  • Do not capitalize modal

page

asset detail page;Search Results page

  • Bold when in steps, e.g., Click the Account page.

  • Italicized when referencing the page name in non-steps, e.g., You can view details on the Account page.

  • Do not capiraliza the word page

Save

Whenever in edit mode, the Save button displays for users to save their updates.

Use the following standard:

Click Save.

Do not use the following:

Click the Save button.

Search bar

Displays next to Syniti logo

 

tab

Tabs display on asset pages.

  • Bold when in steps, e.g., Click the Connections tab.

  • Italicized when referencing the tab name in non-steps, e.g., After a system is created, create a connection on the Connections tab

  • Always lowercase the word tab.

toggle

Switch that can be toggled on and off. Example is on the Create Implementations page, the Include in Data Quality toggle.

This type of control is toggled on or off.

Example:

Set the Include in Data Quality toggle to off to . . . .

Help icon

Displays next to fields, toggles, and other controls for field level help

 

Page help icon

Displays on the Site menu; opens page-level help when clicked

 

Page-level help

Displays when the user clicks the Page help icon in the Site toolbar

 

Notifications icon

The bell icon on the Site menu

 

filter

Filters appear on the left of asset detail pages and the search results page to further restrict records.

 

Jump To links

Accessed after clicking in the Search bar, these links display as the asset names that will return a search on that asset.

 

breadcrumbs

Displays at top of pages indicating the path taken through related assets

Do not capitalize the word breadcrumb.
