SKP What's New Standards

Types of release notes should be ordered as follows:

  1. New Features

  2. Enhancements

  3. Bug Fixes

New Features and Enhancements

Include “Refer to” statement and a link to relevant use case in release notes announcing enhancements or new features.

For example:

Fields can now be deleted from a System. Fields that have been deleted from a System appear as Deprecated on the Fields tab of the Data Set detail page. A filter for Depreciated fields has been added to the Data Set detail page on the Fields Tab.

Refer to Import fields to a System and Set up Data Sets for more information.

To show that the enhancement was made for our users, to make their lives easier:

  • Explain how the enhancement will make working in the SKP easier and/or better for the user.

  • Make the release note in active voice to make it a little more exciting, and it gives us the opportunity to name ourselves (we, at Syniti) as the hero of this tiny little release story.

  • If providing steps to finish a task, use this imperative structure: "To do this thing, complete this step." For example, "To copy the Tenant ID to your clipboard, click the Tenant name."

 

Bug Fixes

Keep notes for bug fixes simple and limited to one sentence. If the way in which the issue was fixed is a change for the user, you can mention it in the note with “by.”

Include the Jira ticket in brackets at the beginning of the release note.

The date you use for a bug fix release note should be the day that the Jira ticket status was moved from Dev -> Prod. When a ticket is moved to Prod in Jira, it means that the code is live in production. For one-off bug fixes that are not part of a feature release, this is most often the date on which the bug fix will be visible to users.

For example: It might help the reader to know that the related asset name is now being truncated.

[KYB-1054] – Fixed an issue where long asset names were causing a horizontal scroll bar for related assets on the asset detail page by truncating the name of the related asset.

The reader does not necessarily need to know that the misalignment was fixed by resizing a div container.

[KYB-1157] - Fixed an issue on the Add Fields tab of the Data Set detail page where the Add icon’s plus sign was misaligned.
