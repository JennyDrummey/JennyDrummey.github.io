SKP Use Case Example

Sponsors

Sponsor assets

Sponsors review assets through their lifecycle to ensure that assets are as valuable to the organization as possible. When users update assets, the Syniti Knowledge Platform notifies the sponsors assigned to those assets. Sponsors can either endorse or contest the asset on the asset detail page after each update. If more than half of the assigned sponsors endorse an asset, the asset is accepted.

If any user edits an asset that is in Review or Accepted status, the asset is moved to Contested status. Once the endorsement threshold is met, which is when the majority of sponsors have endorsed the asset, the asset is then moved to an Accepted status. The action taken by each sponsor is indicated in the Sponsors panel on the asset detail page.

Sponsorship statuses are:

  • Candidate— Asset has not been moved to Review. Details are still being defined.

  • Review— Asset is under review by sponsors. All sponsors and assignees are notified when the asset is in review.

  • Accepted— Endorsement threshold met and the asset is accepted. All sponsors are notified when the asset is accepted.

Sponsors can perform the following actions for assets:

  • Endorse— Click Endorse in the endorsement notification.

  • Contest—

    1. Click Contest in the endorsement notification.

    2. Enter a comment explaining contestation; this field is required.

    3. Click Done.

      {% include note.html content=“This changes the asset’s status back to Candidate. A sponsor must start the process over by sending the asset for review.” %}

  • Send for Review— Click Send for Review in the sponsorship status area.

To start the endorsement process, click the Send For Review button in the Status panel.

Add sponsors to assets

To add sponsors to an asset:

  1. Click the Plus(+) icon next to Sponsors.

    {% include note.html content=“If there are already existing sponsors assigned to an asset, the + icon displays as the Edit icon.” %}

  2. Select one or more users in the Add Sponsors pane.

  3. Click Ok; the Add Sponsors pane closes.

  4. Click Save.

To remove sponsors from an asset:

  1. Click the Edit icon.

  2. Click the x icon next to the user name(s) in the Add Sponsors pane to remove the users from the selection list.

  3. Click Ok; the Add Sponsors pane closes.

  4. Click Save.

Contest assets as a non-sponsor

Users that are not assigned as sponsors can contest assets that they think should be improved. If an Accepted asset is contested, it is moved back to Candidate status and must be re-endorsed by the asset’s sponsors. When assets are contested, all sponsors and assignees are notified by email (if allowed by their individual user settings). Refer to Sponsor Assets for more information.

To contest an asset:

  1. Click the … icon for the asset.

  2. Click Contest [ Asset ].

  3. Enter a comment explaining contestation; this field is required.

  4. Click Done.
