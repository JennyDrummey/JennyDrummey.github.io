Use Case Guidelines

The following examples are the ways in which the SKT use case standards differ from the SST standards:

  • Use bold text (only) for options (links, icons, etc) that are a recipient of a user action in a use case

  • Italicize page names

  • There are no Page Descriptions

Release Notes Guidelines

  • Use the following headings to categorize release notes:
  • New Features
  • Enhancements
  • Bug Fixes

  • If there is more than one entry for a category, use bullets.

  • Only write about tickets that pertain to the UI
