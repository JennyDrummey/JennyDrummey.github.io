Workflow - Peer Review

  1. Peer reviewer adds feedback to the Author Notes document comparing the changes in that document with the dev environment.

The Peer Reviewers reads for typos, clear language and confirms that standards are applied. They also test links in the document and, if possible, test the steps as well. Review Check List for adding a Use Case and Check List for Adding Field Descriptions for more information.

  1. As a result of that feedback, the Author makes the updates and merges this branch to the develop_nnn/develop/Staginginnn branch, and builds in jenkins in the dev environment (and prod environment as needed).

  2. Author posts in the doc_checkouts channel that the changes are complete.

Note: Use the @doc command in Slack to call attention to questions or feedback needed.
