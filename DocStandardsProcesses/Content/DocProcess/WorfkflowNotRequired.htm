Worfkflow - Not Required

A Jira ticket does not require documentation if it is:

  • Related to a Veracode Scan for the SAP PQ

  • A research or investigative ticket

  • A highly complex ticket that contains information that the user doesn't need to know

  • An Epic (SST) or Mission (SKT) (and the documentation is attached to the individual tickets)

  • A bug found only in a dev environment. For tickets of this type, the Affected Environments field displays Development only.

If the ticket does not need to be documented, in Jira:

  1. Click the Documentation Status field.

  2. Select Not Required.

NOTE: Leave your name in the Writer Assigned field.
