Workflow - Not Started

Dev/PM/Others Add Tickets Manually in Jira

The default Documentation Status setting when a Jira ticket is created is Not Started.

Technical Writers track all documentation content in JIRA, including:

  • New features and updates

  • All Bug issues

  • Customer Support issues

  • Other tickets (tasks, defects) that describe changes to behavior or the UI that were previously released.

  • Online Help feedback, tracked on the DocTeam ToDos board. Refer to Incorporating User Feedback.

NOTE: Technical Writers enter issues related to how the online help works, and unrelated to content (as in, typos, broken links, Map ID corrections) on the DocTeamToDos board in JIRA.

The process begins in Jira when PM/Dev move a ticket from the backlog to the DSP70 board for the current release (SST and Install Manual), or when tickets move onto the Kyber board (SKT)
