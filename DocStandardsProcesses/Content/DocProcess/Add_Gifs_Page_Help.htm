Add a Gif or Image to Page-level Help

To add a gif to page-level help:

  1. Create the gif.

  2. Ask an engineer to add the gif to the Syniti CDC.

  3. Add the gif file to the page-level help in i18Nexus.

Add/Update the gif in the Syniti CDN

Gifs are stored on the Syniti CDN, which requires access to AWS. Provide an engineer with these instructions and ask for the file path when the gif is uploaded.

The engineer needs Production AWS Access. That is where the CDN bucket is stored.

To add or update a gif in the Syniti CDN:

  1. Log in to boa-igc-dev.

  2. Switch to devops role for the production account 723806118003.

  3. Go to S3.

  4. Open bucket cdn-syniti-com.

  5. Upload/replace the image to images/ folder.

  6. The image is accessible at https://cdn.syniti.com/images/name-of-file.gif.

NOTE: If you override an image that is currently there it will take ~24 hours for the CDN to purge the contents.

Add the Gif to Page-level Help

Use the following markdown to add the gif to page-level help:

Format: ![gif](FilePathFromEngineering)

Example: ![gif](https://cdn.syniti.com/images/ObjectsSetUsagetoProfiling.gif)

Add an Image to Page-level Help

Images can be stored in the Resources/Images folder in the igc-help Flare project so that they are on the hosted server. You can then reference them to add that image to the page-level help.

  1. In the igc-help project, add the image to the Resources/folder, noting the folder and file name.

  2. Check in the repo and build in dev and prod so the image is accessible.

  3. In I18nexus, in the markdown in the page-help namespace, use this syntax:

:![title on hover]{path to image with image name)

For examle:

![Business process example](https://skthelp.syniti.com/Resources/Images/DQExamples/BusinessConceptExample.png)

After you have saved the change and run the workflow, the image displays in the vercel preview.
