Sources of Content

Changes to the Installation Manuals are tracked in Jira. Sources of changes include:

  • Jira tickets

  • Slack messages

  • Support emails

Organization of the Install Manuals Flare Project

The project is organized by manual. Several sections are used across multiple manuals, with conditional tags applied.

The Version History topic (Appendix A for the Installation Manual and the SRHS Guide) is updated for each manual with all updates made to the manual in the release. Appendix A is not delivered with the final system, it is used so that the documentation team can track changes for each version. The most recent entry is at the top.

Update the Installation Manuals

The following steps describe the collaboration, review, approval and final formatting process to get the Installation Manuals ready to publish for each software release.

Members of the Documentation Team take turns updating the Manuals for each release.

Follow these steps to update the install manuals:

  1. The Tech Writer synchronizes their local current releases branch with the BackOfficeAssoc dsp-install-manuals branch.

    NOTE: If you do not have a local copy of the install manuals on your laptop, see Clone the dsp-install-manual

  2. The TW creates a new Staging branch based on the latest Staging branch in the dsp-install-manuals repo.

    NOTE: If the manual should need additional changes after the Stagingnnn is created, create a new branch that can be merged into the existing Stagingnnn. (e.g., git checkout -b DT1268) This way the new work can be merged through GitHub to the existing Stagingnnn.

  3. The TW creates a Jira ticket in the Syniti Stewardship Tier (DSP70) project for updating the manuals for this release.

  4. The TW links all tickets that describe updates to manuals to this ticket to track the updates.

  5. The TW copies and pastes the sections that need updating into a Google doc, enables Suggestions and makes updates based on the tickets.

  6. The TW adds a link to the Google doc to the ticket, asks PM to review the changes and sets the Documentation Status to Review.

  7. Once the collaboration is complete and the content is final, the approver sets the Documentation Status field to of the installation manual Jira issue to Approved.

  8. The TW makes the changes in the Stangingnnn branch of the Flare project. Refer to Update Online Help for more information. After all changes are made, the TW builds the project in the dev environment in Jenkins.

  9. The TW sets the Documentation Status field of the installation manual Jira issue to Peer Review and requests a peer review from another tech writer on the team in the doc_checkouts Slack channel.

  10. After peer review is complete, if needed, the TW updates the Flare project and builds the install manuals Flare project in Dev and Prod environments in Jenkins.

  11. The TW sends notification that the final manuals are ready to Support with a link to the production version and then closes the Jira ticket.

Clone the dsp-install-manuals repo

To clone the dsp-install-manuals repo, follow the instructions here.

Create a New Staging Branch Based on the Latest Staging Branch

NOTE: You MUST perform these steps each time you checkout a branch to ensure you are working in the latest copy of the master branch.

  1. In Windows-Explorer, right-click your dsp-install-manuals local folder and select Git Bash Here.

    NOTE: If you do not have a local dsp-install-manuals repo, see the section Initial Clone of Project Files for instructions.

    NOTE: If pulling down a branch for the first time, enter the git command git fetch origin <release branch name>, then proceeded with the following steps.

  1. Switch to the Stagingnnn branch where nnn is the latest release, by entering git checkout Stagingnnn.
  2. Enter the git command, git pull origin Stagingnnn
  3. Enter the command git checkout -b <Stagingyyy where yyy is the new SST version name>
  4. Open the project in Flare and update the DSPVersion variable to the current release version.
  5. Open the home.html file and make the follow two updates:
    1. Update the Installation, Upgrade and Requirement links to the new version number.
    2. Add a link under the Previous Versions section to the previous version of the Install manual.
  6. Update the version number in the first TOC item for the InstallHTML5, SRHS GuidesHTML5 and UpgradeHTML5 TOCs.
  7. Save and close Flare, and return to the git bash window.
  8. Enter the command git add ..
  9. Enter the command git commit -m "<your commit message>".
  10. Enter the command git push origin <new branch name>.

Create a JIRA Ticket for the Manuals

  1. Log into JIRA.
  2. Click the Create button on the top menu.
  3. Select the Syniti Stewardship Tier (DSP70) project from the Project dropdown.
  4. Select Document from the Issue Type dropdown.
  5. Enter a name in the Summary field. E.g. "Update the Install and Upgrade Manuals for 7.4.6."
  6. Enter a description in the Description text box. E.g. “Update the Install Manuals and SRHS Guides for the 7.4.6 release" and include a link to the Google doc.
  7. Add the Fix Version as the release number.
  8. Add the Assignee as yourself.
  9. Click Create.

Link Tickets Related to Manuals to JIRA Epic

Add links to the epic to any tickets that affect the installation manual.
