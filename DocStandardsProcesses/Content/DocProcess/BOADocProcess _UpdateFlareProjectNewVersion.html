Update Flare Project for a New SST Version

After a version of the SST has been released, the doc team creates a new branch for the SST help based on the finalized branch of the current help. This new version becomes the help project that is now “in development.”

  1. Right-click your local repo folder (dsp-docs) and select Git Bash Here from the dropdown menu to open the command prompt at that location.

  2. Enter the git command git checkout develop_nnn, where nnn is the version number of the finalized help (as in, the help that was just released).

  3. Enter the git command git pull origin develop_nnn, where nnn is the version number of the finalized help.

  4. Enter the git command git checkout -b develop_NEWDSPVERSIONNUMBER.

  5. Open Flare and add a “What’s New in [New Version Number]” topic.

  6. Update the two What’s New links in the TopNav TOC to point to this new topic, and update the version number in the TOC.

  7. Copy the content from the What's New topic of the last finalized version of help to the top of the topic "Changes in Previous Versions". Add bookmarks to the headings as needed.

  8. Update the Version Variable in the Variable window and on the Variables tab in the target.

  9. Update the sitemap URL with the new version (https://dsphelp.syniti.com/xxx/*) on the Search tab in the target.

  10. Save and close Flare.

  11. Return to git bash.

  12. Stage your updates for commit by entering the command git add .

  13. Commit staged updates by entering the command git commit -m “commit message”

  14. Push your updates to the remote repo by entering the command git push origin develop_NEWDSPVERSIONNUMBER. You do not need to create a PR for this new branch name, as you are adding a new branch to the repo that we will all now be working in.

  15. Build the help in jenkins in dev and prod. This allows the test team to confirm that links between the product version in development and the online help still work.

  16. Post a message to the #doc_checkouts slack channel to let the other writers know that the new branch is available.

  17. Locate the ticket in the Jira project that QE entered to track the testing of the online help, for example DSP70-921.

  18. Mention a QE resource on the ticket, and copy the URL of the prod site into the comment.
