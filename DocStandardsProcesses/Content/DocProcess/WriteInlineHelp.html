Overview of In-line Help

In-line help in the Knowledge Platform is an evolving new feature. The goal is to make all in-app help progressive so that combined, we provide the right level of information to a wide range of audiences (where users can continue to read on if they need more details):

  • In-line Help—provides the smallest chunk of info with a link to static guidance for more information

  • Static Guidance(Page-level Help)—provides a bit more detail than inline help with a link to the online help for more information

  • Online Help—provides the most detailed information we have

To write in-line help for the Knowledge Platform:

  1. Work with the UX team towards the start of the mission to determine where in the product in-line help would be beneficial. Think about

    • What fields need extra explaining?

    • What functionality isn't super intuitive?

  2. Draft content for in-line help using the In-line Help Standards.

  3. Send in-line help to the mission's PM and the UX team for review. Once approved, UX adds the in-line help to the UI mockups.

  4. Make sure the development team has the in-line help content so they can add it to the dev site.

Guidelines

When writing in-line help, follow these guidelines to write quick, simply and conscience prose:

  • Keep the text to a minimum

  • Provide an example, if applicable

  • Write in the second person and with an active voice

  • Include a "Learn more" link to static guidance for more information. (This feature isn't available just yet, but will be soon!)
