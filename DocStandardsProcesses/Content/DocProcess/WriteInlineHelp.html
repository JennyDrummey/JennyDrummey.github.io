Overview of Field Help

Field help in the Knowledge Platform is an evolving new feature. The goal is to make all in-app help progressive so that combined, we provide the right level of information to a wide range of audiences (where users can continue to read on if they need more details):

  • Field Help—provides the smallest chunk of info

  • Page-level Help—provides a bit more detail than field help with a link to the online help for more information

  • Online Help—provides the most detailed information we have

To write field help for the Knowledge Platform:

  1. Work with the UX team towards the start of the mission to determine where in the product field help would be beneficial. Think about

    • What fields need extra explaining?

    • What functionality isn't super intuitive?

  2. Draft content for field help using the Field Help Standards.

  3. Send field help to the mission's PM and the doc team for review.

Guidelines

When writing field help, follow these guidelines to write quick, simply and conscience prose:

  • Keep the text to a minimum

  • Provide an example, if applicable

  • Write in the second person and with an active voice
