Update Static Guidance

PM and the Documentation Team determine when static guidance should be added or updated.

To update static guidance:

  1. Create a JIRA ticket (typically on the DocTeam ToDos board, but sometimes Jon Green creates static guidance tickets on the Kyber board) to track the work to update the static guidance.

  2. Pull down the master branch from ui-component-libs repo.

  3. Create a new branch, using the Jira ticket number as a name.

  4. Navigate to this file: ui-guidance-libs/projects/ui-component-libs/src/lib/guidance/constants/static-guidance.ts.

    NOTE: Open with Atom or Visual Studio Code.

  5. Update the descriptionText as needed, save and close the file.

  6. Navigate to the file ui-component-libs/projects/ui-commponent libs/package.json.

  7. Increment the third decimal place by 1 for version (ex: 1.1.14 to 1.1.15).

  1. Save and push your repo to the ui-component-libs repo.

  2. Open a PR to merge it to master.

  3. Ask the App Engineering Group to review the PR.

  4. Set the ticket to Code Review, if applicable.

  5. Once the changes have passed code review, send the PR to your team’s QE (Karen Dunson) for testing.

  6. When the dev and QE resources have both approved the PRs, merge the PRs to master.

  7. Notify Nathan Sweezeythat changes are ready to be deployed - we’re working to automate this process, but for now there’s still a little bit of manual intervention left

  8. After the changes have been deployed to dev, and have been tested, Nathan Sweezeycan handle the process of moving the changes to preview and/or production

Identifying apps/services in ui-guidance-libs

In each of our projects that need to support contextual guidance, they'll have a file in the app/models folder named guidancecontext.ts. Looking at the appcontext value on line 11 will show what context value the specific application is passing into the guidance component, which is what's used to look up the static guidance.
