Check List Add New SST Component

Complete the following tasks when adding a new component (i.e., a WebApp) to Help.

  1. Add the new component's WebApp groups to the WebApp Groups topic in System Admin

  2. Add a brief description of the component to the general SST Overview topic or the ADM Overview topic if relevant.

  3. If the component has objects that can be CTS'd, add those to the CTS Overview page in System Administration.

  4. Ensure the new component includes the standard sections: Overview, Configuration and Setup for [component name], Set up Security for (new component name). The Security topic should be the first topic in the Configuration and Setup section.

  5. Add the new component's alias file value range to process document.

  6. Add a concept for the new component

  7. Add a snippet for the new component.

  8. Add new component name and pages to the What's New topic under Enhancements.

  9. Add the new component name to the TopNav menu drop-down menu.

  10. Add a condition tag for the new component.

  11. Add a button for the new component to the home page.

    Buttons are alphabetized under sections on the home page. The full version of help contains a section called Master Data Management that the solex version does not.

To add a button to the home page in Flare:

1. In Content Explorer, open the topic home.htm.

2. Copy a button and paste it in the correct location

3. Update the text to the new component name.

4. Edit the hyperlink to open to the Overview section of the new component.

Add a Page Descriptions Navigation page

Each component with page descriptions should have the first menu item in the use case TOC as Component name Page Descriptions. This links to a page with a hyperlinked list of all pages in the compoonent. An alphabetical navigation bar displays at the top. This bar is controlled by javascript, and the scripts have been added to the resources/scripts folder. The scripts were downloaded and slightly modified to suit our needs. To create a page descriptions topic, copy another page descriptions topic and paste it into the new component;s Page_Desc folder. Update as needed for the component.

* Ensure the usecase master page is applied.

* Replace the snippet with the component name

* Update the CT on the topic

* Replace the hyperlinks with the correct pages

* Ensure the menu item has been added to the component's TOC
