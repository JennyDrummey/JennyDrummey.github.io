Create Weekly SKP Release Notes

Tech Writers and PM (Jon, Emily, Andy and Caroline) work together to create the weekly release notes for Syniti Knowledge Platform (SKP).

NOTE: The weekly update is for these systems - SKP, Replicate, and Match (as of Oct ‘22). Both Construct and Migrate produce monthly release notes per their cadence.

At a high level, PM produces the bulk of the release notes by:

  • Pulling the filtered list of PRs each week from the skt-ui repo.

  • Reviewing the list and, using track changes, deleting PRs that are not relevant.

  • Providing details for the PRs where the provided descriptions are not enough.

To start the process, on Friday morning of each week, PM runs the query as described below.

As a PM, to produce the release notes:

  1. Right-click the skt-ui folder for your local repo and select Git Bash Here.

    NOTE: If pulling down this branch for the first time, enter the git command git fetch origin main, then proceed with the following steps.

  2. Enter the git command git pull origin main

  3. Run this command:

    git log --pretty=format:%s --since="7 days ago" | cat

  4. Copy the line above, right click on Git Bash Here and paste, click enter.

  5. Highlight the text in the console, right-click and select Copy.

  6. Create a google doc with the name SKP Release Notes mm/dd/yy.

  7. Paste the console’s list of changes into this document.

  8. Review the list of issues and, using track changes, delete any you think could be excluded.

    NOTE: If it’s not clear what work is associated with a PR from the name alone you may need to copy the description from the PR. See the steps for getting the description from the PR below.

  9. Post a note in the Slack channel #release-notes asking the other PMs for review and feedback.

  10. Once PM feedback has been incorporated, let the Tech Writers know in the #release-notes channel that they are ready to post.

  11. The Technical Writer makes changes to the google doc following the standards listed below.

  12. The Technical Writer posts a note in the #release-notes channel to ask for peer review from other writers before adding the content to help.

  13. After the peer review is done, the Technical Writer adds the weekly updates to SCReleases.htm file in the Flare project igc-help following these steps. Add a new month if needed. Examples are provided.

  14. Once published in Flare, post the release notes in the #skp_releases channel. Example provided below.

How to find PR descriptions

In the skt-repo, click the Pull Requests tab, then click the Closed filter.

Search on the PR number included in the release notes output.

Information about the PR displays on the Conversation tab, including the summary/description.

This needs to be done before EOB on Friday and posted to #skp_releases.

Standards for Weekly Release Notes for SKP

  • Asset names are lower case

  • Page names are italicized

  • Only the release note title and colon are bold - nothing else

  • When applicable, remove the Jira ticket number and engineer's name from the PR titles

  • Mentions of i18nexus with “translations” are not included in the weekly notes.
