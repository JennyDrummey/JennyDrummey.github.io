Create Weekly SKP Release Notes

Tech Writers and PM (Jon, Emily, and Caroline) work together to create the weekly release notes for Syniti Knowledge Platform (SKP).

NOTE: The weekly update is for these systems - SKP, Replicate, and Match (as of Oct ‘22). Both Construct and Migrate will produce monthly release notes per their cadence once they are active.

  • The PM will pull the filtered list of PRs each week.

  • The PM will review the list and line through any not relevant.

  • The PM will provide details for the PRs where the provided descriptions are not enough.

To start the process, on Friday morning of each week, a query is run as described below.

To pull the filtered list of release notes::

  1. Right-click the skt-ui folder for your local repo and select Git Bash Here.

    NOTE: If pulling down this branch for the first time, enter the git command git fetch origin main, then proceed with the following steps.

  2. Enter the git command git pull origin main

  3. Run this command:

    git log --pretty=format:%s --since="7 days ago" | cat

    **Copy the line above, right click on Git Bash Here and paste, click enter.

  4. Highlight the text in the console, right-click and select Copy.

  5. Create a google doc with the name SKP Release Notes mm/dd/yy.

  6. Paste the console’s list of changes into this document.

  7. Review the list of issues and line through any you think could be excluded.

    NOTE: If it’s not clear what work is associated with a PR from the name alone you may need to copy the description from the PR. See the steps for getting the description from the PR below.

  8. Post a note in the Slack channel - # release notes to share the google doc link with all participants (Doc Team and Dylan and PM)

  9. The PM makes changes to the google doc and posts a note in the same Slack channel - # release notes to show that it is complete and ready to load to the online help.

  10. Update the release notes to meet the standards listed below.

  11. Add the weekly updates to SCReleases.htm file in the Flare project igc-help following these steps. Add a new month if needed. Examples are provided.

  12. Once published in Flare, post the release notes in the #skp_releases channel. Example provided below.

How to find PR descriptions

In the skt-repo, click the Pull Requests tab, then click the Closed filter.

Search on the PR number included in the release notes output.

Information about the PR displays on the Conversation tab, including the summary/description.

Post the Release Notes link to Slack channel

This needs to be done before EOB on Friday and posted to #skp_releases.

Standards for Weekly Release Notes for SKP

  • Asset names are lower case

  • Page names are italicized

  • Only the release note title and colon are bold - nothing else

  • When applicable, remove the Jira ticket number and engineer's name from the PR titles

  • Replace mentions of i18nexus with “translations”
