Create Weekly SKP Release Notes

Tech Writers and PM (Jon and Caroline) work together to create the weekly release notes for the SKP.

To start the process, on Thursday of each week, the Tech Writer runs a query as described below.

To create the release notes:

  1. Right-click the skt-ui folder for your local repo and select Git Bash Here.

    NOTE: If pulling down this branch for the first time, enter the git command git fetch origin main, then proceed with the following steps.

  2. Enter the git command git pull origin main

  3. Run this command:

    git log --pretty=format:%s --since="7 days ago" | cat

  4. Highlight the text in the console, right-click and select Copy.

  5. Create a google doc with the name SKP Release Notes mm/dd/yy.

  6. Paste the console’s list of changes into this document.

  7. Review the list of issues and highlight any you think could be excluded, including a comment asking if we can remove it.

    NOTE: If it’s not clear what work is associated with a PR from the name alone you may need to copy the description from the PR,. See the steps for getting the description from the PR below.

  8. Share a link to the google doc with the release notes for review to PM (Jon G/Andrew S)

  9. PM returns the list of notes with a brief description.

  10. Add the weekly updates to the SCReleases.htm file in the Flare project following these steps. Once published in Flare, post the release notes in the #skp_releases channel.

How to find PR descriptions

In the skt-repo, click the Pull Requests tab, then click the Closed filter.

Search on the PR number included in the release notes output.

Information about the PR displays on the Conversation tab, including the summary/description.
