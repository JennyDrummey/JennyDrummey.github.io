Troubleshoot Localization PRs

When Chromatic Fails

Chromatic defines all the pieces of the UI. It’s a library of all the UI elements, both what it looks like in the UI and the code behind it. If a visual change has been made to one of these defined pieces, Chromatic fails the localization PR.

We can easily view the change in Chromatic, accept that this is a change we want, and merge the PR.

This is how you know Chromatic failed your PR:

To accept the change and continue with the PR:

  1. Click Details; CircleCi opens.

  2. Copy and paste this link into a new browser.

    NOTE: In this case, the Subject Matter Experts label, which is a label defined in Chromatic, was modified

  3. Click Review change to see what the before and after are.

  4. Click Accept if you meant to make this change. Otherwise, click Deny.

  5. Return to Chromatic and run the build from fail; rerunning the build takes a few minutes.

  1. Return to the PR once the build successfully ran. Confirm that all checks have passed.
