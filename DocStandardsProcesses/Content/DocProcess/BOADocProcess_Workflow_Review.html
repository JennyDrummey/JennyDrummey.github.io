Workflow - Review

PM may review documentation on the Jira ticket, if it is a release note or a small update. For new features and significant updates, TWs create a google doc for PM to review.

When the content is ready for review:

  1. Access the ticket in Jira.

  2. Update the Documentation Status field to Review.

  3. @ mentions PM in a comment on the ticket asking for a review.

PM confirms that new and updated features and resolved issues are described completely and correctly from a user’s perspective.

Either:

  • PM approves the content by setting the Documentation Status field to Approved. Refer to Workflow - Approved for more information.

Or

  • PM notes on the Jira ticket what needs to be updated, and the process begins again at the In Process step.
