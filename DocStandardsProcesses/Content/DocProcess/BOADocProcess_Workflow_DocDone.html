Workflow - Doc Done

Build help in the dev/prod environment in Jenkins

If the updates to help are for a version that has been released, build the help in dev and then in prod.

If the updates to help are for a version that has not been released, you can build in dev as needed.

Refer to Update the Help on Dev and Prod sites for steps to push the changes to the dev and prod sites as needed.

Change status

Once content has passed peer review and has been pushed to prod/dev as needed:

  1. Change the Documentation Status list box on the Jira tickets listed in your Author Notes to Doc Done in JIRA.

  2. Leave your name assigned to these ticketsWriters Assigned field.
