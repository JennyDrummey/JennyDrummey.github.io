Prepare SST Release Notes

NOTE: We stopped producing a pdf version of the release notes with version 7.4.3. I am keeping this topic in case we have to produce them again. I will delete it in a few months if we don't hear complaints that require us to start making the pdfs again. NO NEED TO REVIEW THIS TOPIC UNLESS WE REINTRODUCE THIS PROCESS.

NOTE: The Syniti Solutions release notes are produced every time there is a release of the SST. The SAP Accelerators release notes are produced when the version of the SST will go through the SAP PQ process.

The content of the What’s New topic in the help is prepared as a Word document and distributed as a secured PDF via the support site. It is stored in a Published folder, along with the Installation and Configuration manuals, for each released version of the SST.

As the content in the release notes is based on the finalized content of the What’s New topic in the help, ensure the steps to review and prepare the content have been completed. Refer to Prepare What’s New Topic for Release for more information.

To create the Word version of the release notes:

  1. Open the last published release notes document in Word. Published documents are stored in Sharepoint here.

  2. Save the release notes document with the updated version number locally.

  3. Open the Document Properties and update the Category to update the version number in the title page and footers.

    NOTE: If you are running the Office 360 version of Word, access Document Properties via File > Info > Properties > Advanced Properties and click the Summary tab in the dialogue box to access Category.

  4. Update the Software Release Date on the cover page.

  5. Replace the content of the release notes with the content from the What’s New topic.

  6. Format the Release Notes in Word using the following styles: Body Style, and Headings 1, 2 and 3.

  7. Add hyperlinks from the release notes to the referenced help topics .NOTE: In the release notes, add in the online help to references. For example, in the What’s New topic: Refer to Add a Request for more information. In the release notes: Refer to Add a Request in the online help for more information.

    NOTE: Add a hyperlink from the reference in the release notes to the topic in the correct version of help (Syniti Solutions v. SAP Accelerators and version number). Use Word’s Add Hyperlink feature, and be sure to link to the production site, not the development site.

  8. Update the TOC in Word. Click in the TOC. Press F9. Select Update entire table in Update Table of Contents dialogue box.

  9. Upload the release notes Word document to the appropriate version folder in the Published folder on Sharepoint.

  10. Create a Jira ticket for the release notes, and link it to the ticket for the installation manuals.

  11. Add a link to the ticket to the Word draft of the release notes.

  12. Assign the ticket to PM for review by assigning the Documentation Status of Review and mentioning the PM in the comments.

  13. After PM approves the document, create the PDF following the steps in Convert Word to PDF topic.

  14. Save the Word and PDF to the correct version folder in the Published folder on SharePoint

  15. Send a link to the secured PDF to support.

  16. Close the Jira ticket related to the release notes and change the Documentation Status to Doc Done.

  17. Apply any changes from PM’s review to the What’s New topic.

IMPORTANT: If changes are made to the install manuals, tell Support that new versions are available.

Compare Content in Word (What’s New topic v. Release Notes)

NOTE: These steps are optional and intended for situations where multiple updates must be made to the What’s New topic/release notes document after the release notes document has been finalized.

The Release Notes document and the What’s New topic in the help are updated frequently during final review prior to help delivery. Compare the content to ensure that it is identical, if needed.

The results of a compare can show discrepancies that are easy to miss with a side-by-side review.

NOTE: The ‘Refer to . . .’ references are different. In the Release Notes, they include ‘in the online help.

To compare Release Notes and What’s New content after significant updates:

  1. Save the Release Notes document with " _compare" appended to the file name.

  2. Generate a Word document from Flare with the What’s New topic (select the appropriate Condition Build Expression)

  3. In both documents, delete the cover page, toc, and introduction section. Also delete the headings. (You just need the text to compare). Save the changes.

  4. Open one of the documents in Word. (It does not matter which one)

  5. On the Review tab, click Compare and select Compare two versions of a document.

  6. In the Compare Documents window, browse to and select your Original and Revised documents. (Release notes can be the original or the What’s New; it does not matter).

  7. Click More, and then select/deselect the options as needed to match what is shown in the following screen shot:

  8. Click OK.

  9. Save the resulting Word document.
