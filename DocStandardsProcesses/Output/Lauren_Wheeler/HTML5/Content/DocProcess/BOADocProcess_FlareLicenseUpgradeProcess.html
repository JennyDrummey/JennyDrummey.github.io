MadCap Flare Licensing and Upgrade Process

NOTE: The Documentation Team uses Madcap Flare to generate the online Help for Syniti products. Contact the Technical Writer Manager for the Flare license key to install or upgrade the software and the maintenance key to use to report problems with the Flare software.

When upgrading Flare, you must upgrade your local version and the upgrade the version on the Jenkins server.

Upgrade your local version

When upgrading the software, the Technical Writer Manager assigns one writer to upgrade and test the upgrade for Flare. Upgrade your local version of Flare and perform the following checks:

  1. Build each Target in each Flare project to make sure there are no build errors.

NOTE: If there are build errors export the build log and report the issue to Flare support.

  1. Check the output to make sure that the master pages are applied correctly.

  1. Click on all clickable items and verify they work as expected (for example the arrow drop down menu next to the logo in the header).

  2. Verify that the feedback link in the topic footer works.

  3. Make sure the print and Remove Highlights icons in the Help toolbar work.

For the SST Help project if everything checks out, commit a build to source control and make sure that the Context Sensitive Help is working as expected in a test SST environment.

Once the build is confirmed working in the SST test environment, notify the Documentation Team that they can upgrade.

Upgrade Flare on the Jenkins Server

Upgrading Flare on the Jenkins server requires an AWS VPN connection and a login to the remote desktop.

Get the correct access

  • AWS VPN—Submit a ticket to Syniti Support to request a connection to the AWS VPN. You will receive an email from CloudOps with instructions on what to download and how to connect. Run through these steps to ensure you can connect.

  • Remote Desktop—Ask a member of the Doc Team to create you a login. Refer to Create a User Account for the Remote Desktop for details. Log into the Remote Desktop to ensure you can connect. IP Address of the server is 172.31.88.245

Perform the Flare upgrade

  1. Submit a ticket to Syniti Support, requesting CloudOps to take an AMI the Jenkins server before performing the Flare upgrade. Syniti Support will need the IP Address of the server to complete this request (172.31.88.245).

    NOTE: An AMI is an image; we want to capture the state of the server prior to the upgrade for safety. If anything breaks, we can revert to a copy of the server beforehand.

  1. Wait for CloudOps to reply saying that the AMI finished.

  2. Message the team to ensure no one is actively using Flare.

  3. Connect to the AWS VPN. If the connection was correctly set up, this step is as simple as opening the AWS VPN Client and clicking Connect.

  4. Log into the Remote Desktop. If the connection was correctly set up, this step is as simple as opening the Remote Desktop Connection and clicking Connect. If you did not save your login credentials, you will have to enter them before connecting.

  5. Open Flare and perform the upgrade.

    NOTE: If downloads from the Flare website are blocked on the server, copy the Flare install .exe file from your local computer to the Jenkins server. Then, open and run the .exe file on the server to complete the install.

  6. Reply in the Support ticket that the upgrade is complete and request another AMI of the server to capture the new version of Flare.

Create a User Account for the Remote Desktop

For new team members, create a user account for the remote desktop.

To create a user account for the remote desktop:

  1. Connect to the AWS VPN.

  2. Log into the Remote Desktop.

  3. Right-click Windows Start and select Computer Management.

  4. Expand Local Users and Groups.

  5. Right-click Users and select New User…

  6. Enter a User name following the convention FirstInitialLastName, e.g., EWarden.

  7. Enter any password (the user can change it at login). Feel free to use a password generator to take the guesswork out of the process.

  8. Uncheck User must change password at next logon check box.

  9. Check Password never expires check box.

  10. Click Create.

  11. Return to Computer Management.

  12. Click Groups.

  13. Right-click Administrators and select Add to Group …

  14. Click Add.

  15. Enter the username in the Enter the object names to select text box and click Check Names. The new account name displays.

  16. Click OK. The user has been created and added as an Administrator.
