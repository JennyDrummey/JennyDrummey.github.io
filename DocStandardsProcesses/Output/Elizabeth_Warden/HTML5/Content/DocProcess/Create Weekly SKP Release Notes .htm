Create Weekly SKP Release Notes

Tech Writers and PM (Jon, Emily, Andy and Caroline) work together to create the weekly release notes for Syniti Knowledge Platform (SKP).

NOTE: The weekly update is for the SKP, Migrate, Construct and Orchestrate. Other modules are note yet released. Both Construct and Migrate produce monthly release notes per their cadence.

At a high level, PM produces the bulk of the release notes by:

  • Pulling the filtered list of PRs each week from the skt-ui repo.

  • Reviewing the list and, using track changes, deleting PRs that are not relevant.

  • Running a query in Jira to see which issues were tested successfully in prod. The release notes should only include these issues.

  • Providing details for the PRs where the provided descriptions are not enough.

To start the process, on Friday morning of each week, PM runs the query as described below.

As a PM, to produce the release notes:

  1. Run the following query: project="KYB" AND "Test Status[Labels]"="ProdTested". Refer to Run a Query in Jira for more information.

  2. Review the tickets that have been updated last and compare them to what was listed in the previous week's release notes.

  3. Draft the release notes in a Google doc with the name SKP Release Notes mm/dd/yy.

  4. Post a note in the Slack channel #release-notes asking the other PMs for review and feedback.

  5. Once PM feedback has been incorporated, let the Tech Writers know in the #release-notes channel that they are ready to post.

  6. The Technical Writer makes changes to the google doc following the standards listed below.

  7. The Technical Writer posts a note in the #release-notes channel to ask for peer review from other writers before adding the content to help.

  8. After the peer review is done, the Technical Writer adds the weekly updates to SCReleases.htm file in the Flare project igc-help following these steps. Add a new month if needed. Examples are provided.

  9. Once published in Flare, post the release notes in the #skp_releases channel. Example provided below.

Run a Query in Jira

To run a query in Jira:

  1. Click in the Search bar.

  2. Click Advanced issue search.

  3. Click Switch to JQL.

  4. Delete any contents that may be in the text box.

  5. Paste the query into the text box.

  6. Click Search.

Post the Release Notes link to Slack channel

This needs to be done before EOB on Friday and posted to #skp_releases.

Example posting:

Weekly release notes for 7/1/2022 for the SKP are published.

https://skthelp.syniti.com/SCReleases.htm

 

Standards for Weekly Release Notes for SKP

  • Asset names are lower case

  • Page names are italicized

  • Only the release note title and colon are bold - nothing else

  • When applicable, remove the Jira ticket number and engineer's name from the PR titles

  • Mentions of i18nexus with “translations” are not included in the weekly notes.
